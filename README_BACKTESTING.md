# Historical F&O Backtesting & Data Ingestion Guide

This pipeline allows you to download, reconstruct, and backtest end-of-day options data for the National Stock Exchange (NSE). The framework consists of two main tools: the `historical_loader.py` engine for ingestion, and the `backtest_runner.py` for simulated trading.

## 💾 Where is the data stored?

All heavily reconstructed data is persisted locally in your SQLite database located at:
`backend/scanner.db`

The data is logically split across several different tables within the database. The most important tables for backtesting are:
1. **`market_snapshots`**: This is the master table containing over 60+ reconstructed option chain and greek features for every single trading day, mapped to the underlying indices and top stocks. The data specifically populated by the historical downloader is marked with `data_source = 'EOD_HISTORICAL'`.
2. **`iv_history`**: A separate table that explicitly tracks the day-by-day average ATM Implied Volatility for every symbol. This is what the scanner relies on to compute the `IV Rank` features during testing (it reads backwards over a 1-year history rolling window).

If you are querying manually via a Python shell or SQLite viewer engine, a sample command to view the records is:
```sql
SELECT snapshot_time, symbol, score, signal, trade_result 
FROM market_snapshots 
WHERE data_source="EOD_HISTORICAL"
```

## 🚀 Available Commands

### 1. The Ingestion Engine (`historical_loader.py`)

This engine handles downloading the raw files, unzipping the payloads, mapping the new vs old NSE data layouts, building the options chains, computing exact limits like Max Pain, Net Gamma Exposure, PCR ratios, and Black-Scholes Greeks, and finally dumping it into the DB.

* **`python backend/historical_loader.py download`**: Exclusively connects to the internet to download all the missing ZIP and Spot Price data locally to `./historical_data`.
* **`python backend/historical_loader.py process`**: Reconstructs the mathematical features using downloaded data only, keeping everything in memory and exporting a final un-committed raw `.csv` file. 
* **`python backend/historical_loader.py load-db`**: Commits the processed CSV generated by `process` directly into the `backend/scanner.db`. It handles internal Idempotency (`INSERT OR REPLACE` logic) so you can run it identically 100 times without ever duplicating existing data rows.
* **`python backend/historical_loader.py full`**: Executes the entire 3-step pipeline sequence automatically (Download → Process → Load-DB) to get you up and running smoothly.
* **`python backend/historical_loader.py status`**: A lightweight sanity command that just reports how many historical rows currently exist inside the Database, alongside average signal scores to verify successful loads.

*You can filter any command by passing parameters like `--start 2024-01-01 --end 2024-06-05`*

### 2. The Backtest Engine (`backtest_runner.py`)

Once data is loaded, you interact with the offline backtesting suite. The backtester mathematically simulates executing the trades based on strict entry rules, holding to expiry/SL/TP conditions, and evaluating Expectancy (Profit Margins). 

* **`python backend/backtest_runner.py`**: Runs a standard one-shot simulation loop utilizing default parameters (Score >= 75, Confidence > 0.5, TP at +40%, SL at -25%).
* **`python backend/backtest_runner.py --score 40 --confidence 0`**: Runs a one-shot simulation deliberately dropping the strict filtering parameters to catch more marginal/weaker trades for debugging volumes.
* **`python backend/backtest_runner.py --signal BULLISH`**: Strict focus filters, forcing the backtester to only consider taking "Long Calls / Short Puts" signals. Discards all Bearish/Neutral signals regardless of score.
* **`python backend/backtest_runner.py --regime EXPIRY`**: Forces test permutations over specific NSE market scenarios (Trendings, Pinned limits, or Gamma Squeezes). Excludes options not identified matching the regime conditions.
* **`python backend/backtest_runner.py --optimise`**: Engages Grid Search mode! It iteratively re-runs the simulation matrix hundreds of times stepping through 0-100 values to locate the ultimate "Top 10 Parameter Combinations" sorted internally by their mathematical Expectancy per-trade percentage.

#### Understanding The Parameters Array:
`--tp` / `--sl`: Float numbers acting as percentages for your stop limits. `--tp 50` means a +50% limit order Take Profit.
`--score`: Internal ML threshold from 0 to 100.
`--symbols`: Only focus on distinct subsets like `--symbols NIFTY,BANKNIFTY`.
